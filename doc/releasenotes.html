SSC Version 267 (SAM version 2021.12.02, PySAM version 3.0.0)

  • Battery model:
    • Refactor the battery dispatch options such that forecast and dispatch algorithm are seperate options. New options for batt_dispatch_choice are:
      • Front of meter:
        • 0 - Economic Dispatch
        • 1 - PV Smoothing Dispatch
        • 2 - Custom Dispatch
        • 3 - Manual Dispatch
      • Behind the meter:
        • 0 - Automatic Peak Shaving Dispatch
        • 1 - Input Grid Power Target
        • 2 - Custom Dispatch
        • 3 - Manual Dispatch
        • 4 - Price Signals Dispatch
    • Forecast is now controlled by new variables batt_dispatch_wf_forecast_choice and batt_dispatch_load_forecast_choice
      • 0 - Perfect look ahead
      • 1 - One day look behind
      • 2 - Custom (using batt_pv_clipping_forecast and batt_pv_ac_forecast for weather and batt_load_ac_forecast and batt_load_ac_forecast_escalation for load
    • New behind the meter dispatch and power flow variables including:
      • batt_dispatch_auto_can_clipcharge - clip charging for DC connected batteries
      • batt_dispatch_auto_btm_can_discharge_to_grid - discharging to grid for custom dispatch
      • batt_dispatch_charge_only_system_exceeds_load - For AC connected batteries, 0 means the battery can charge while the grid meets the load. (1 is prior behavior)
      • batt_dispatch_discharge_only_load_exceeds_system - For behind the meter batteries - 0 means the system discharge the battery while generation is sold to the grid. (1 is prior behavior)
      • dispatch_manual_btm_discharge_to_grid - a matrix specifying the schedule of when manual dispatch can discharge to the grid
    • New grid outage variables including:
      • grid_outage - array of timesteps when the grid is unavailable (1) or available (0)
      • batt_minimum_outage_SOC - minimum SOC variable used when grid_outage[t] is 1
      • crit_load_escalation - array of annual escalation factors for critical load
    • en_standalone_batt - automatically specifies an array of zeroes for gen
  • BatteryStateful model:
    • New cycle_counts variable, a 2xn matrix used for rainflow counting when batt_life is set to 0.
  • UtilityRate5 model:
    • Front of meter configurations (Equpartflip, Levpartflip, Merchantplant, Saleleasback, Singleowner) now use the utility rate module to compute grid charging and parastics costs
    • ur_ec_enable_billing_demand is now ur_enable_billing_demand
    • ur_ec_billing_demand_lookback_percentages is now ur_billing_demand_lookback_percentages
    • ur_ec_billing_demand_lookback_period is now ur_billing_demand_lookback_period
    • ur_ec_billing_demand_minimum is now ur_billing_demand_minimum
    • New variable ur_dc_billing_demand_periods, specifying for which demand charge periods billing demand is calculated
  • Front of meter financial configurations:
    • New variables including:
      • en_electricity_rates - specify whether the system pays for battery charging or system loads (parasitcs) using a retail rate (1) or the PPA price (0)
      • om_land_lease
      • om_land_lease_escal
      • uland_area
